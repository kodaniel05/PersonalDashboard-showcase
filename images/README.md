# Images

This folder is reserved for public-safe visual assets.

## Screenshot Guidelines

- Use sanitized demo data only.
- Do not include real personal information, account data, documents, API keys, addresses, client data, or private notes.
- Prefer screenshots that show workflow and interface quality rather than sensitive content.
- Keep filenames stable so README links continue to work.

Recommended screenshots:

- `screenshots/dashboard-home.png`
- `screenshots/productivity-view.png`
- `screenshots/finance-view.png`
- `screenshots/personal-view.png`
- `screenshots/relationships-view.png`
- `screenshots/explore-view.png`
- `screenshots/document-vault.png`
- `screenshots/live-market-watchlist.png`
- `screenshots/daily-snapshot.png`
- `screenshots/planning-tools.png`

## Diagram Guidelines

Architecture diagrams can be exported into `diagrams/` if needed. Keep diagrams high-level and avoid private schemas, exact endpoint details, or implementation-specific algorithms.
